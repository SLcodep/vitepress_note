在前端进行性能分析时主要使用的几个工具是：Performance、Lighthouse、network。
根据京山桥米这个项目对请求头、响应头，Performance，Lighthouse有一个更深的了解。

## network
### 请求报文分析
![请求标头](/public/qian/请求标头.png)

在上面这个图中能看到很多常见的关键词：

- **Method+URL**
确认请求的资源是什么、用的什么HTTP方法(get,post等)
示例：`GET /img/bg0.c6e55986.png`

- **Cache-Control**
`Cache-Control:no-cache`:**不直接使用本地缓存**，需要先向服务器验证资源是否更新（协商缓存），验证通过后才能用缓存。
`Cache-Control: public, max-age=3600` 表示资源可以被缓存，并且在1小时内有效。

- **Accept-Encoding**
作用:查看浏览器支持的压缩格式（Gzip/Brotli 等），判断服务器是否做了压缩优化。
示例:`Accept-Encoding: gzip, deflate`,支持 Gzip 压缩

- **User-Agent**
识别客户端设备/浏览器类型等

### 响应报文分析

- **Status Code**
作用:判断请求是否成功，快速定位错误类型（2xx 成功、4xx 客户端错、5xx 服务端错）
如，`200 OK → 成功`；`404 Not Found → 资源不存在`

- **Access-Control-Allow-Origin**
用于控制哪些域名可以跨域请求当前资源，解决浏览器的同源策略限制。
示例：`Access-Control-Allow-Origin:`,公开接口，允许所有域名跨域。

- **缓存体系详解**
强缓存`Cache-Control(HTTP/1.1)`+`Expires(HTTP/1.0)`,Cache-Control优先级更高。
协商缓存`ETag`+`Last-Modified`。

- **Content-Length**
响应体的字节数,浏览器知道需要下载多少字节，可显示加载进度；验证响应体是否完整。
示例，`Content-Length: 7029732`

- **Content-Type**
响应体的 MIME 类型，告诉浏览器 “如何解析该资源”。
HTML 页面   - `Content-Type: text/html; charset=utf-8`
CSS 样式表  - `Content-Type: text/css`
JS 文件     - `Content-Type: application/javascript`
图片        - `Content-Type: image/webp`
JSON 接口响应 - `Content-Type: application/json`
